Core Prompt Design
{
  "ai_summary": "2-3句中文摘要，概括核心内容和亮点",
  "tags": ["LLM应用" | "Agent" | "效率工具" | ...],
  "pm_insight": "产品经理视角的一句话洞察"
}
设计要点：角色设定 → 限定标签范围防止自由发挥 → pm_insight可留空降低幻觉风险 → 严格JSON输出便于程序解析

Thinking
产品思考
🎯
需求 ≠ 功能
在 AI 日报工具开发中，最初想「接入所有渠道」，实际使用后发现非AI内容是噪声。功能做减法、数据做过滤，比堆砌数据源更重要——这与产品设计中「不是所有需求都要满足」是同一个道理。
✍️
Prompt 是产品设计
Prompt 的迭代过程与写 PRD 高度相似：定义角色、规定输出格式、设置边界条件、处理异常。第一版输出不稳定，经三轮迭代才收敛——这让我深刻理解了为什么 Prompt 工程是 AI 产品的核心能力。
🔬
真实使用是最好的测试
把自己作为工具的真实用户，发现「PM视角」字段有时输出废话，于是改了 Prompt 约束。GIS 背景让我对「数据质量即产品质量」有切身理解——在中汽创智的实习中，标注数据质量直接影响了算法模型效果。
